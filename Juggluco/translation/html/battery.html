Keep functioning

Normally when an android device is not used, it enters doze mode to use less energy. In this mode an application can't use TCP connections to exchange data. This means that two android devices can only communicate when both screens are on. You can exempt an application from this restriction by disallowing 'Battery optimization' in android settings' menu. Update in doze mode, shows this setting and changing it opens the android setting menu where you can change this setting. On some phones only apps already exempted are shown and to add an app you have to go to android settings->battery->app battery management and select Juggluco and turn on allow background activity.

The Bluetooth connection with the sensor is not strictly forbidden by Android and will normally also work without specifying an exemption. Problems can arise after some time or when Juggluco has not been in the foreground for a while. This can be different for different versions of android and for different device manufacturers.

Some device manufacturers have imposed additional restrictions on applications running in the background or starting at boot, see https://dontkillmyapp.com.

An app frequently in the foreground is turned off by Android later than an app not in the foreground. I received a report from someone with an Samsung phone who had turned off all visible notifications of Juggluco, using it only to send glucose values to xDrip, which had connection problems with the sensor that were solved after turning the visible signs of Juggluco on again.
