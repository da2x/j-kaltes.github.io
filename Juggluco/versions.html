(Use cat or copy as shown below to make one apk out of these files.) As far as I know you can only install an apk on a WearOS watch by using adb from a computer. In Developer options of your watch you should enable "ADB debugging" and probably "Debug over WIFI". And on you phone press something like (and confirming the connection on you watch screen); 

adb connect YOUR_WATCH_IP:5555

adb -s YOUR_WATCH_IP:5555 install Juggluco2.3.13-wear.apk

You can also install Juggluco for WearOS via Google Play. At least, I can find Juggluco by searching for Juggluco in Google Play on my watch.

This is a version of Juggluco that can run on Wear Os. After installation on a Wear Os, Juggluco on a mobile phone should initiate the app by sending the sensor data to the watch. There are two possibilities:

1. The sensor keeps contact with the smartphone and glucose values are sent to the watch by IP/TCP.
2. The direction is changed after the data are sent to the watch.

All can be done under Left Menu-> Watch -> Wear Os in the mobile phone version of Juggluco.
First time sending of all data to the watch takes a lot of time.

On my Samsung galaxy watch4 you can turn off "mobile battery saver" in the developer options, which are normally hidden. To see them you have to go to "About Watch"-> Software-> "Software version" and tap 7 times on "Software version". Thereafter, you can find "developer options" in the settings above "Software update". I don't know if it has any effect.
